前置知识

一、
在 PHP 5 中，assert() 是一个函数，我们可以用 $_=assert;$_() 这样的形式来实现代码的动态执行。但是在 PHP 7 中，assert() 变成了一个和 eval() 一样的语言结构，不再支持上面那种调用方法。（但是好像在 PHP 7.0.12 下还能这样调用）

PHP5中，是不支持 ($a)() 这种调用方法的，但在 PHP 7 中支持这种调用方法，因此支持这么写 ('phpinfo')(); #解释了吃瓜杯 shellme的疑惑

二、
任何字符与 0xff 异或都会取相反，这样就能减少运算量了。

三、
注意：测试中发现，传值时对于要计算的部分不能用括号括起来，因为括号也将被识别为传入的字符串，可以使用 {} 代替，原因是 PHP 的 use of undefined constant 特性。例如 ${_GET}{a} 这样的语句 PHP 是不会判为错误的，因为 {} 是用来界定变量的，这句话就是会将 _GET 自动看为字符串，也就是 $_GET['a']。${_GET}{%ff} 后面那个 () 为的是能够动态执行传入的 PHP 函数。

四、

最开头的 ?> 闭合了 eval() 函数自带的 。{} 里面包含的 PHP 代码可以被执行
